Preparation Guide

Ace Your Pyrefly Assessment

Everything you need to know before you start. Read this guide carefully — it will directly impact your score.

📋 Assessment Overview
Round 1

Aptitude & Logic Test

20 randomised questions spanning four cognitive domains. Each question is multiple-choice with exactly one correct answer.

⏱ 30 minutes 📝 20 questions 🔒 Fullscreen required
Round 2

Technical Coding Challenge

One real-world coding problem evaluated by AI on logic, code quality, and efficiency. Write clean, working code.

⏱ No hard limit 💻 Any language 🤖 AI graded
🛡️ Anti-Cheat Rules
  • 1
    Fullscreen is mandatory The test only starts after you enter fullscreen. Exiting fullscreen at any point will immediately terminate your session.
  • 2
    No tab switching Switching to another browser tab or window will trigger automatic termination. Open all reference material before you begin.
  • 3
    Window must stay in focus Clicking outside the browser window (e.g. to Alt+Tab to another app) will be detected and may terminate the test.
  • 4
    One attempt per session Each test session is unique and can only be submitted once. Refreshing the page or closing the tab ends the session.
  • 5
    Copy & paste is disabled Right-click context menus and keyboard shortcuts for copying text are disabled during the aptitude round.

💡 Strategy & Tips
⚡

Don't Overthink

Your first instinct is usually correct on pattern-based questions. Spend max 90 seconds per question.

🗂️

Answer All Questions

There is no penalty for wrong answers. Always take a guess — even a random pick has a 25% chance of being right.

📐

Eliminate First

Narrow choices down by eliminating obviously wrong options before committing to an answer.

⏳

Manage Your Time

With 30 minutes for 20 questions, that's 90 seconds each. The timer turns yellow at 10 min and red at 5 min.

🔁

Review if Time Allows

Use the Prev button to go back and revisit flagged questions before submitting.

🖥️

Coding: Readable Code

The AI evaluator rewards clean variable names, comments, and correct logic over brevity.

✅ Do's & Don'ts

✅ Do

  • ✓ Use a stable internet connection
  • ✓ Take the test in a quiet environment
  • ✓ Read each question completely before answering
  • ✓ Use scratch paper for numerical calculations
  • ✓ Write complete, runnable code for the coding round
  • ✓ Check your code logic before submitting

❌ Don't

  • ✗ Switch tabs or windows mid-test
  • ✗ Exit fullscreen during the aptitude round
  • ✗ Refresh or close the browser tab
  • ✗ Spend more than 2 minutes on a single question
  • ✗ Submit without answering all questions if you have time
  • ✗ Leave the coding editor blank — even pseudocode is scored
